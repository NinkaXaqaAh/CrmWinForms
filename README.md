# CrmWinForms

CRM для малого бизнеса. WinForms MDI-приложение на **.NET 10**, написано в духе Clean Architecture с разделением на слои Core / Infrastructure / WinForms.

## Возможности

- Клиенты - физлица и компании с реквизитами (ИНН, контактное лицо, адрес, заметки), статусы (Лид / Активный / VIP / Неактивный / Заблокирован), фильтр и поиск.
- Сделки - воронка из 6 этапов (Новая -> Квалификация -> Предложение -> Переговоры -> Выиграна / Проиграна) с суммой, вероятностью и датами закрытия. Доменный сервис валидирует допустимые переходы между этапами.
- Активности - звонки, встречи, письма и задачи. Привязка к клиенту/сделке, приоритеты, отметка «Завершить», подсветка просроченных.
- Товары и услуги - справочник для каталога малого бизнеса (название, артикул, категория, цена, активность).
- Пользователи - учётные записи с ролями (Admin / Manager / Viewer). Управление доступно только Admin'у. PBKDF2-хеширование паролей (100k итераций SHA-256).
- Дашборд - 6 карточек ключевых метрик (выручка, прогноз, просрочки), круговая диаграмма «клиенты по статусам» с процентами, столбчатый график выручки по месяцам.
- Глобальный поиск (Ctrl+K) - одно окно с агрегацией по 4 сущностям, debounce 300 мс.
- Темы - светлая и тёмная, переключаются через меню «Вид -> Тема» с перезапуском.
- Экспорт в Excel - кнопка на каждом списке (Клиенты / Сделки / Активности / Товары / Пользователи), сохраняет числа/даты/деньги нативными Excel-типами для корректной сортировки.
- PDF-отчёт по сделке - одна страница A4 с параметрами, клиентом, описанием и связанными активностями.
- 3-state сортировка- клик по заголовку столбца: А-Я -> Я-А -> исходный порядок. Работает для текстов, чисел и дат.
- Авто-обновление - дашборд и списки сами перечитывают данные при возврате с диалога редактирования.

## Архитектура

| Слой | Роль |
| --- | --- |
| `CrmApp.Core` | Доменные модели, enum'ы, абстракции репозиториев и сервисов, FluentValidation-валидаторы |
| `CrmApp.Infrastructure` | JSON-репозитории, доменные сервисы (Auth, DealPipeline, Dashboard), сидер, ClosedXML/QuestPDF |
| `CrmApp.WinForms` | Формы (MDI), Composition (DI), темизация, глобальный поиск |

JSON-данные хранятся в `%APPDATA%\CrmApp\` (см. `appsettings.json` → `JsonStorage:DataFolder`). Атомарная запись через временный файл + опциональный `.bak`.

## Технологии

- C# 12, .NET 10, WinForms (MDI)
- Microsoft.Extensions.* 10.x - DI, Configuration, Logging, Hosting
- FluentValidation 11.x - валидация
- LiveChartsCore.SkiaSharpView.WinForms 2.x - графики на дашборде
- ClosedXML - экспорт в Excel
- QuestPDF (Community) - PDF-отчёты
- xUnit + coverlet - тесты
- Central Package Management (`Directory.Packages.props`) - версии пакетов в одном месте

## Запуск из исходников

```bash
dotnet restore
dotnet build
dotnet run --project src/CrmApp.WinForms
```

Демо-логины (создаются при первом запуске, если `users.json` пуст):
- `admin` / `admin` — администратор
- `manager` / `manager` — менеджер

## Сборка standalone-exe

Для запуска без установленного .NET Runtime:

```bash
publish.cmd
```

Скрипт делает три вещи:
1. `dotnet publish` в Release с `PublishSingleFile=true` и `--self-contained` — получится один `CrmApp.exe` ~72 MB.
2. Копирует exe в `%LOCALAPPDATA%\CrmApp\CrmApp.exe`.
3. Создаёт ярлык `CRM.lnk` на рабочем столе.

После запуска exe не требует .NET - всё уже внутри.
